Introduction

We are now going to verify a few metrics and present some simple and user-friendly tools to ensure our blog's validity, performance, accessibility and SEO. Those topics are potentially infinite and highly specialized, that is why we will just touch on each of them with automated tools.

Lighthouse

A good way to quickly and simply check webpages performances is by using Lighthouse. It is available in the DevTools of Chromium-based web browsers, which means it can be used when developping on localhost. It is also available as an online service through PageSpeed Insights, but it means that the page must be hosted online already.

Lighthouse provides metrics and advices about performance, accessibility, SEO and other best practices, and it can run in both desktop or mobile configuration. Do not underestimate the mobile devices because more than half of webpage are browsed from mobile device per statcounter.

Accessibility

When it comes to accessibility, WAVE and axe Tools are very nice ways to automatically check how your webpage perform. A very simple way to use it is to install the WAVE and axe DevTools browser extensions. They both runs for local and online page and provides plenty of good advices and checks such as color contrast.

Code validity

For code validity, you probably know already that the Polyglot Tech Blog template allows checking HTML and CSS validity using W3C's vnu-jar. However, W3C maintains plenty of other useful tools that can be used to check yor code validity. Let's mention the internationalization and feed checkers, just to quote some that are applicable to the Polyglot Tech Blog.

Conclusion

We have seen different tools to assess and improve your code in terms of validity, performance, accessibility and SEO. But in the end, all these elements are closely related: invalid code is not properly parsed, leading to performance issues, and then it is not well managed by accessibility tools and badly referenced by search engines. These tools just provide different point of views and advices about your code. They are complementary and, since they are all automated tools, they have some limit in what they can detect.
