In this post, we are going to see what to do to get Polyglot Tech Blog starter running with your content

Add or remove languages

Removing a language is simply deleting the corresponding directory and updating the src > _data > data.json file accordingly. When adding a new language, there are a bit more things to consider, so you can consult the dedicated article.

Add static pages

The starter comes with four static page: "Home", "Stats", "Matches" and "404". They are written in HTML and can be found in each language directory:

src
 ├── (all files in src)
 ├── en
 │   ├── index.html <- the "home" page
 │   ├── about.html
 │   ├── blog.html
 │   └── 404.html
 └── fr
     ├── index.html <- the "home" page
     ├── about.html
     ├── blog.html
     └── 404.html

You may decide to add some static pages. Here as well, there is a dedicated article that covers the topic.

Add posts

This is probably the most basic thing expected from a blog start. You can do this by following the instructions on the dedicated article

Check markup validity

The starter template comes with commands and dependencies to check HTML and CSS validity using W3C's vnu-jar. However, vnu-jar checker is written in Java, so you will have to install a Java Runtime Environment to make these commands work. I recommend the free and excellent Eclipse Temurin.
You can then check the HTML validity with:

npm run check:markup:unix # with npm on Linux and unix-like systems
yarn check:markup:unix # with Yarn on Linux and unix-like systems

npm run check:markup:win # with npm on Window systems
yarn check:markup:win # with Yarn on Windows systems

You can then check the CSS validity with:

npm run check:css:unix # with npm on Linux and unix-like systems
yarn check:css:unix # with Yarn on Linux and unix-like systems

npm run check:css:win # with npm on Window systems
yarn check:css:win # with Yarn on Windows systems
